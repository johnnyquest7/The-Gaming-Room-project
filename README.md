# The-Gaming-Room-project
 
#Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

  A project focused on a software design document, created for the hypothetical client called The Game Room.
The code itself is a group of backend classes that help define, authenitcate, and track game assets.

#What did you do particularly well in developing this documentation?

  I did a lot of my research by reading documentation provided by the companies themselves. I feel like I condensed down a lot of information into
  something thats easier to digest. 

#What about the process of working through a design document did you find helpful when developing the code?

  Referencing the UML diagram was something I had to do alot, to make sure I understood the relation between the 
  classes I was coding. This simple but effective method helped more than I thought it would. 
  
#If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

  I would go back to the Development Tools table and revise it to be more appealing. I would also try and cite more sources
  to supplement the validity of my reccomendations. 

#How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

  The plan set out by the UML coupled with just becoming familiar and reading through all of the code and documents was an essential start to 
  thinking through the design. Its important to consider what common problems a user might need to solve, and what the user is setting out to do
  with the program. The entire process needs to be geared towards its intended group of users.

#How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

  By focusing on one task at a time, and making sure I approach each task with an even amount attention. It's extremely easy to begin making changes 
  to many files and lose track of what you are doing. Desinging software in future projects I would always read up on the documentation for all libraries
  I would be using.
